# 홍시표T 첨삭 — 클로드 데스크탑 워크플로우 (API 0원)

> 이 문서를 **클로드 데스크탑 프로젝트 지침**에 붙여넣거나 첨부하면, 매번 일관된 홍시표T 첨삭이 nonsul 웹앱 DB에 바로 저장된다. Gemini API 호출 없이 클로드 구독으로 첨삭한다.

## 사전 조건
- nonsul MCP 서버가 연결돼 있어야 함 (`claude_desktop_config.json`에 등록 완료).
- 첨삭할 시험지가 웹앱에 등록돼 있어야 함 (`list_exams`로 확인).

## 첨삭 절차 (반드시 이 순서)

1. **방법론 로드** — `correction_guide` 툴을 먼저 호출한다. 톤(존댓말)·채점비중(독해 40~50%)·코멘트 4유형(praise/improvement/error/suggestion)·margin_comments 마킹규칙·출력 스키마를 그대로 따른다.

2. **시험 컨텍스트 로드** — 어떤 시험인지 확정한다.
   - 모르면 `list_exams`로 목록 확인 → `get_exam(exam_id)`로 지문·문제·채점기준(rubric) 전부 로드.
   - 채점기준이 없으면 가이드의 기본 채점비중을 적용한다.

3. **답안 읽기** — 학생이 채팅에 첨부한 답안 이미지(원고지/필기)를 읽어 본문을 전사한다. 여러 장이면 순서대로 이어붙인다.

4. **첨삭 수행** — 가이드대로 꼼꼼하게. 거의 모든 문장에 코멘트, 잘한 곳 구체적 칭찬, 문제 있는 곳 번호 매긴 대안. 채점기준 기반으로 점수 산출.

5. **답안 레코드 생성** — `create_student_answer`:
   - `exam_id`(필수), `student_name`(또는 `student_id`), `question_number`, `answer_text`(전사본)
   - → 반환된 `answer_id`를 다음 단계에 사용.

6. **첨삭 저장** — `save_correction`:
   - `answer_id`, `exam_id`,
   - `answer_outline`(전개 요약 2~3문장), `summary`(종합총평), `strengths`, `improvements`,
   - `scores`(아래 스키마), `total_score`, `grade`,
   - `margin_comments`(아래 스키마), `status: "completed"`
   - → 저장되면 웹앱 `/corrections`에서 빨간펜 뷰어로 바로 확인 가능.

7. **(선택) 학부모 문자 초안** — `create_message(type: "첨삭완료", ...)`로 결과 요약 문자 초안 생성.

## 출력 스키마 (save_correction에 넣을 값)

```json
{
  "answer_outline": "학생이 어떤 논리로 전개했는지 2~3문장",
  "margin_comments": [
    {
      "id": "c1",
      "page": 1,
      "y_position": 0.18,
      "para": 0,
      "quote": "마킹할 정확한 구절(문단 내 유일 식별)",
      "text": "코멘트 내용(존댓말)",
      "type": "praise | improvement | error | suggestion"
    }
  ],
  "scores": [
    {
      "question_number": 1,
      "point_scores": [{ "name": "제시문이해", "earned": 8, "max": 10, "notes": "..." }],
      "deductions": [{ "name": "주술호응", "deduction": 2, "reason": "..." }],
      "subtotal": 25
    }
  ],
  "total_score": 75,
  "grade": "B+",
  "summary": "구조 진단 + 수준 평가 + 핵심 개선점 + 칭찬 + 다음 과제",
  "strengths": "잘한 부분 정리",
  "improvements": "개선 필요 부분 정리"
}
```

- `type`은 반드시 4종 중 하나. `y_position`은 0.0~1.0. `para`는 문단 인덱스(0부터), `quote`는 그 문단의 정확한 구절 → 웹앱 원고지 PDF에 마킹이 그려진다(praise=밑줄+체크, improvement=물결, error=동그라미, suggestion=점선밑줄).

## 사용 예시

> 사용자: "경희대 사회 1008 시험 3번 문항, 김서연 학생 답안이야. 첨삭해줘." (+ 답안 이미지 첨부)
>
> 클로드:
> 1. `correction_guide` 호출
> 2. `list_exams(university:"경희대")` → exam_id 확인 → `get_exam(exam_id)`
> 3. 첨부 이미지에서 답안 전사
> 4. 홍시표T 방법론으로 첨삭
> 5. `create_student_answer(exam_id, student_name:"김서연", question_number:3, answer_text:"...")` → answer_id
> 6. `save_correction(answer_id, exam_id, ...첨삭결과, status:"completed")`
> 7. "첨삭 완료. 웹앱 /corrections에서 확인하세요. 총점 75/B+. 학부모 문자 초안도 만들까요?"

## 일괄 첨삭
여러 학생 답안을 한 번에 받으면, 학생별로 3~6단계를 반복한다. 각 학생마다 `create_student_answer` → `save_correction`. 마지막에 반 전체 점수 분포를 요약해준다.
