# 프로세스논술 수업자료·해설지 — 클로드 데스크탑 워크플로우 (API 0원)

> 이 문서를 클로드 데스크탑 프로젝트 지침에 붙여넣으면, 시험지를 올렸을 때 프로세스논술 실물과 동일한 해설지·수업자료가 nonsul 웹앱 DB에 바로 저장된다. Gemini가 "개좆같이" 뽑던 걸 클로드 구독으로 일관되게 대체한다.

## 핵심 원칙
- 해설지/수업자료는 **프로세스논술이 하는 것을 그대로 자동화**한다. 내 독학 방법론이 아니라 **프로세스 첨삭교육론**을 따른다.
- 생성 전에 반드시 `explanation_guide`를 먼저 호출해서 5단계 구조·발문→사고력 분류·기초/심화 득점포인트·문체규칙을 로드한다.

## 절차

1. **방법론 로드** — `explanation_guide` 호출.

2. **시험 컨텍스트 확보** — 두 경로:
   - 이미 웹앱에 있으면: `list_exams` → `get_exam(exam_id)`로 지문·문제·채점기준 로드.
   - 새 시험이면: 사용자가 PDF/텍스트/사진을 채팅에 올린다 → 직접 읽어 제시문(라벨+전문)·문제(번호+발문+분량+배점)·표/그래프를 정확히 전사한다. (한 글자도 빠뜨리지 말 것. 표는 마크다운으로.)

3. **논제분석** — 모든 문제를 한눈에 보는 개요표: 출제유형 / 제시문 배치 / 핵심키워드 / 분량. 각 발문을 4대 사고력(분석적/추론적/종합적/대인적) 중 하나로 분류한다.

4. **제시문분석** — 제시문마다: 핵심논지(1~2문장) / 핵심키워드 / 다른 제시문과의 관계(대립·보완·인과·기준-적용) / 사고방식.

5. **문제해결** — 문제마다: 답안 구조(첫째/둘째/셋째, 두괄식) / 접근법 / 비교는 기준설정·변별·추론적비교, 적용은 기준설정·세분화·연결대상 구체화, 비판은 4대 비판툴(역효과·고비용·목적배반·대비가치) 명시.

6. **채점기준표** — 기초 득점포인트(질문호응/빠뜨리지않기/두괄식/분량분배/공통주제/대립쌍 변별) + 심화 득점포인트를 항목으로, 배점·체크리스트·감점요소·☆핵심포인트.

7. **예시답안** — 분량 조건에 정확히 맞춘 만점 수준 답안, 글자수 명시. 문체: 답안은 '~이다/~한다'체, 제시문 베끼기 금지(재진술), 마크다운 강조 금지.

8. **저장** — `save_material(exam_id, type, content, brand)`:
   - `type`: "해설지" | "채점기준표" | "ppt"
   - `content`: `explanation_guide`의 출력 스키마(overview/passage_analyses/solutions/scoring_criteria/model_answers) 그대로.
   - `brand`: "프로세스" 또는 "독립"
   - → 웹앱 시험 상세 페이지에서 바로 확인 가능.

## 문체 규칙 (절대 준수)
- 해설은 '~이다/~한다'체(해라체), 존댓말·AI투("~할 수 있습니다") 금지.
- 마크다운 **bold**/*italic* 금지. 강조는 작은따옴표('이렇게')만.
- 제시문 문장 그대로 베끼지 말고 재진술. '관점'은 반드시 "무엇을 중요시하는가"로 도출.

## 수업 PPT (type: "ppt")
- 검은 배경 + 흰 텍스트, 제시문 1문단=1슬라이드, 표/그림 별도 슬라이드, 문제는 마지막에 각 1슬라이드.

## 일괄
여러 회차를 받으면 회차마다 2~8단계를 반복하고, 끝에 회차별 난이도·유형 분포를 요약한다.
